# STM32 TOTP Hardware Authenticator & Server

Аппаратно-программный комплекс двухфакторной аутентификации. Проект состоит из прошивки для микроконтроллера STM32 и управляющего сервера на Python. Система реализует защищенный обмен данными, генерацию одноразовых паролей (TOTP) и криптографическую проверку подлинности устройства.

## Логика системы

1. **Безопасная инициализация** (`SET_KEY`): Сервер генерирует случайный 128-битный секретный ключ, шифрует его по алгоритму AES-128-ECB и передает на плату. Микроконтроллер расшифровывает его и сохраняет в Flash-память.
2. **Синхронизация времени** (`SET_TIME`): Сервер передает текущую метку времени (Timestamp) на устройство.
3. **Проверка TOTP** (`GET_TOTP`): Плата и сервер независимо друг от друга рассчитывают 6-значный одноразовый пароль на основе времени по алгоритму TOTP (HMAC-SHA256). Если коды совпадают — первый этап проверки пройден.
4. **Защита от подмены** (Challenge-Response через `GET_HMAC`): Сервер отправляет случайную строку (вызов). Плата подписывает её секретным ключом с помощью HMAC-SHA256 и возвращает ответ. Это гарантирует, что к порту подключено именно ваше устройство, а не эмулятор.
5. **Индикация доступа**: При успешной проверке на плате загорается зеленый светодиод (`SET_GREEN`), при ошибке — красный (`SET_RED`).

## Структура проекта

* `C code/` — Исходный код прошивки для микроконтроллера STM32 (HAL-библиотеки, работа с Flash, USB CDC и криптографические алгоритмы).
* `server.py` — Основной Python-скрипт сервера для взаимодействия с устройством и верификации.

##  Требования и технологии

* **Железо**: Плата на базе микроконтроллера STM32 (например, STM32L5 или аналогичная) с поддержкой USB-периферии.
* **Прошивка**: Язык Си (STM32Cube HAL). Реализованы кастомные функции SHA-256 и HMAC без внешних тяжелых библиотек.
* **Сервер**: Python 3.10+, менеджер зависимостей Poetry.
